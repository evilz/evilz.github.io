Cet épisode du workshop Hot Chocolate est consacré à le rôle du middleware dans Hot Chocolate pour enrichir ou contrôler le pipeline GraphQL. Bumble dev avance pas à pas pour montrer comment construire une API GraphQL propre, tout en expliquant les choix de modélisation et les impacts côté implémentation.

La série est intéressante parce qu'elle ne se limite pas à la syntaxe : elle parle aussi d'architecture, de conventions et de performance. Chaque épisode apporte une brique supplémentaire pour mieux comprendre Hot Chocolate et construire une API GraphQL plus robuste avec .NET.
